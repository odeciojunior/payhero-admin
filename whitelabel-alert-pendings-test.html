Changes Made:

  • Updated resources/modules/global/css/global.css to use var(--primary, #2e85ec) instead of hardcoded #2e85ec
  • Added CSS rules to WhitelabelController.php to generate the alert-pendings-box-2 styling in dynamic CSS
  • The alert box now automatically adapts to each whitelabel client's primary color
  • Maintains backward compatibility with fallback to original color

How it works:

The .alert-pendings-box-2 class now uses background-color: var(--primary, #2e85ec) which means:

  1. First, try to use the --primary CSS variable (set by whitelabel system)
  2. If not available, fall back to the original blue color #2e85ec
  3. Each client's primary color is automatically applied to the alert pendings box
